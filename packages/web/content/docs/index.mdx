## What is ClawRun?

ClawRun is a hosting and lifecycle layer for open-source AI agents. It deploys agents into secure Firecracker microVMs and manages their full lifecycle:

- **Startup** — provision a sandbox, install the agent, start the daemon
- **Keep-alive** — heartbeat monitoring, TTL extension based on activity
- **Snapshot/Resume** — automatic snapshotting on idle, instant resume on new messages
- **Wake-on-message** — webhook-driven wake from any connected channel

## Quick Start

Install the CLI globally:

```bash
npm install -g clawrun
```

Deploy your first agent:

```bash
clawrun deploy
```

Send a message:

```bash
clawrun agent -m "Hello, agent!"
```

## Architecture

ClawRun follows a **persistent sandbox** model. The agent daemon runs continuously inside a sandbox. ClawRun manages the sandbox lifecycle around it — extending TTL when active, snapshotting when idle, and waking on incoming messages.

The agent runs **inside** the sandbox, not in the serverless function. The function is the orchestrator. The sandbox is where the agent lives.

## Key Concepts

- **Agent** — the AI agent running inside the sandbox (e.g., ZeroClaw)
- **Sandbox** — an isolated microVM managed by a provider (e.g., Vercel Sandbox)
- **Sidecar** — a supervisor process inside the sandbox that manages the agent daemon
- **Channel** — a messaging platform connected via webhooks (Telegram, Discord, Slack, etc.)
- **Preset** — a declarative configuration bundle (agent + provider + defaults)
